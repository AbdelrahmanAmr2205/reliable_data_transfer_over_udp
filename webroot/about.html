About This Server

Alexandria University — CC451 Computer Networks Lab 4, Spring 2026

What is this?

A hand-built HTTP/1.0 server that does not use the OS TCP stack. Instead it runs a custom RDTSocket class over raw UDP that reimplements the core TCP mechanisms in Python user space.

Implemented features

  • Three-way handshake — SYN → SYNACK → ACK
  • Stop-and-Wait reliable data transfer (RFC 1122 §3.4.1)
  • 16-bit ones-complement checksum (RFC 1071)
  • Packet-loss simulation — configurable drop probability
  • Packet-corruption simulation — random bit-flip
  • Retransmission on timeout
  • Duplicate-segment detection and re-ACK
  • Sequence numbers and cumulative acknowledgements
  • FIN / ACK connection teardown
  • HTTP/1.0 GET and POST methods
  • 200 OK, 201 Created, 403 Forbidden, 404 Not Found, 405, 500
  • TCP ↔ RDT bridge so real browsers can connect

Files

  • packet.py — wire format, flags, checksum
  • rdt_socket.py — reliable transport over UDP
  • http_handler.py — HTTP/1.0 request & response parsing
  • http_server.py — HTTP application server
  • http_client.py — HTTP application client
  • tcp_bridge.py — TCP ↔ RDT proxy for browser access
  • tests.py — test suite (packet, RDT, HTTP layers)
